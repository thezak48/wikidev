이 페이지에서는 Prowlarr에서 인덱서를 추가하고 구성하는 방법에 대해 설명합니다.

# 인덱서 추가하기

인덱서를 추가하려면 먼저 왼쪽의 `인덱서`를 클릭한 다음 페이지 상단의 <kb>+</kb> `인덱서 추가`를 클릭하세요.

![ind_1_addindexer.png](/assets/prowlarr/ind_1_addindexer.png)

목록에서 인덱서를 선택하거나 상자에 부분 이름을 입력하여 인덱서를 찾을 수 있습니다. 인덱서가 목록에 없는 경우 "일반 Newznab" (Usenet용) 또는 "일반 Torznab" (토렌트용)을 사용할 수도 있습니다. 그렇지 않은 경우 [인덱서 요청 사이트](https://requests.prowlarr.com/)를 방문하세요.

> `일반 Newznab` 또는 `일반 Torznab`을 사용하려면 인덱서가 표준화된 *znab 형식을 지원해야 합니다. 그렇지 않으면 작동하지 않을 수 있습니다.
{.is-info}

> 참고: 거의 모든 Usenet 사이트는 일반 Newznab과 호환됩니다.
{.is-warning}

> 트래커 또는 인덱서가 목록에 없고 [지원되는 인덱서](/prowlarr/supported-indexers) 페이지에 없는 경우 [인덱서 요청 사이트](https://requests.prowlarr.com)를 통해 추가 요청할 수 있습니다.
{.is-info}

# 인덱서 편집하기

인덱서를 편집하려면 먼저 왼쪽의 `인덱서`를 클릭한 다음 편집하려는 인덱서 오른쪽 끝에 있는 렌치 아이콘을 클릭하세요.

# 인덱서 ID 또는 URL 보기

인덱서에 대한 세부 정보를 보려면 먼저 왼쪽의 `인덱서`를 클릭한 다음 인덱서 이름 열의 인덱서 링크를 클릭하세요. (이전에는 오른쪽의 (i) 아이콘)

사용 가능한 세부 정보는 다음과 같습니다:

- Prowlarr에서의 ID
- 설명
- 인코딩
- 언어
- 사이트
- Newznab/Torznab Prowlarr URL
- 사이트 기능

# 인덱서 설정

인덱서를 선택하면 구성에 필요한 자세한 정보가 포함된 팝업이 표시됩니다. 특정 인덱서의 필수 필드 및 구성 중인 인덱서 유형에 따라 특정 설정이 약간 다를 수 있음에 유의하세요.

![ind_3_indexer2.png](/assets/prowlarr/ind_3_indexer2.png)

- 이름 - 이 인덱서에 대한 이름을 선택하세요. 앱과 동기화될 때 뒤에 `(Prowlarr)`이 추가됩니다.
- 활성화 - 이 인덱서를 활성화하려면 확인란을 선택하세요.
- 리디렉션 - 리디렉션이 필요한 경우 확인란을 선택하세요. 리디렉션이 필요한 인덱서는 몇 개뿐입니다. 이를 통해 차단을 피할 수 있습니다. 활성화된 경우 이는 Prowlarr를 통해 프록시하지 않고 직접 애플리케이션으로 그랩 링크를 전달합니다.

> 리디렉션은 일반적으로 특정 인덱서에만 필요합니다.
{.is-info}

- 동기화 프로필 - 여기에서 동기화 프로필을 선택하세요. 이는 [`설정` => `앱`](/prowlarr/settings#applications)에서 생성할 수 있습니다. 표준 기본 프로필은 이미 존재하며 다음과 같습니다:

> 여러 인덱서 인스턴스를 생성하여 앱별로 다른 설정을 사용할 수 있습니다.
{.is-info}

![ind_3_settingsapps.png](/assets/prowlarr/ind_3_settingsapps.png)

- URL - Prowlarr에서 사용할 URL을 선택하세요. 비어 있으면 기본/첫 번째 URL이 사용됩니다.
- 다운로드 링크 - 토렌트 인덱서를 추가하는 경우 사용할 다운로드 링크 유형을 선택해야 할 수 있습니다.
- (고급 옵션) API 경로 - 인덱서의 API 경로입니다. 일반적으로 `/api`입니다.
- 자격 증명 - 많은 인덱서와 트래커에서 인증/로그인을 요구합니다. API 키, RSS 키, 세션 ID, 쿠키 또는 기타 인덱서의 자격 증명(일반적으로 프로필 페이지나 보안 아래에서 찾을 수 있음)을 입력해야 할 수 있습니다. 특정 인덱서에 대한 검색 순서 또는 기타 옵션을 선택해야 할 수도 있습니다.
  - API 키
  - RSS 키
  - 세션 ID
  - 쿠키
  - 사용자 이름/비밀번호
  - 기타
- (고급 옵션) 추가 매개변수 - 이 인덱서에 대한 요청에 추가할 매개변수입니다.
- VIP 만료 - 만료 1주 전에 알림을 받으려면 ISO 형식(yyyy-MM-DD)으로 날짜를 입력하세요. 그렇지 않으면 비워 두세요.
- 태그 - 태그를 사용하여 기본 다운로드 클라이언트, 인덱서 프록시, 애플리케이션별 인덱서 또는 인덱서를 구성하는 데 사용하세요.
- (고급 옵션) 쿼리 제한 - 인덱서가 API 호출 수를 제한하는 경우 이곳에 숫자를 입력하여 제한을 초과하지 않도록 할 수 있습니다.
- (고급 옵션) 그랩 제한 - 인덱서가 하루에 그랩 수를 제한하는 경우 이곳에 숫자를 입력하여 제한을 초과하지 않도록 할 수 있습니다. 그랩 제한에 도달하면 추가 쿼리는 \*Arr 앱에서 처리되지 않는 예외를 발생시킵니다. 다른 앱은 다를 수 있습니다.
- (고급 옵션) 시드 비율 - 토렌트 인덱서 전용: 토렌트가 중지되기 전에 도달해야 하는 비율입니다. 비워 두면 앱의 기본값이 사용됩니다.
- (고급 옵션) 시드 시간 - 토렌트 인덱서 전용: 중지되기 전에 토렌트를 시드해야 하는 시간입니다. 값은 분 단위입니다. 비워 두면 앱의 기본값이 사용됩니다.
- (고급 옵션) 인덱서 우선 순위 - 릴리스 우선 순위 결정 시 이 인덱서를 다른 인덱서보다 우선시할 우선 순위입니다. 1은 가장 높은 우선 순위이고 50은 가장 낮은 우선 순위입니다. 이러한 우선 순위는 \*Arr 앱에 동기화됩니다.

- 인덱서를 테스트하고 녹색 확인 표시가 나타나면 저장할 준비가 된 것입니다. 저장하면 동기화 설정에 따라 자동으로 앱에 추가됩니다.

## 사용자 정의 YML 정의 추가하기

- YML 정의는 일시적으로 캐시되며 개발 목적으로 변경한 경우 캐시를 기다리거나 Prowlarr를 다시 시작해야 합니다.
- 지원되지 않는 인덱서에 대한 사용자 정의 Cardigann 호환 YML 정의 파일을 추가하거나 기존 정의를 테스트하려면 다음을 수행하세요:
  - [Prowlarr의 앱 데이터 폴더](/prowlarr/appdata-directory)의 `Definitions` 폴더 내에 `Custom`이라는 이름의 사용자 정의 인덱서 정의 폴더를 탐색(또는 생성)하세요.
    - 예시 경로:
      - Windows: `C:\ProgramData\Prowlarr\Definitions\Custom`
      - Linux: `/home/$USER/.config/Prowlarr/Definitions/Custom`
      - OSX: `/Users/$USER/.config/Prowlarr/Definitions/Custom`

  - [Cardigann 호환 YML 파일](/prowlarr/cardigann-yml-definition)을 폴더에 저장하고 Prowlarr가 액세스할 수 있는 권한이 있는지 확인하세요.

> 정의의 파일 이름과 ID는 고유해야 하며 다른 기존 정의와 충돌해서는 안 됩니다. 정의의 이름도 고유하게 설정하는 것이 좋습니다.
{.is-info}
